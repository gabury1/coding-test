# 2025-12-24 코딩테스트 문제 풀이

## 해결한 문제

### [7576] 토마토 ✅
- **알고리즘**: BFS (너비 우선 탐색)
- **핵심 개념**:
  - `collections.deque`를 사용한 BFS 구현
  - 큐에 (y, x, days)를 저장하여 날짜 추적
  - 초기에 모든 익은 토마토를 큐에 넣고 시작
  - 방문 처리(`grid[y][x] = 1`)로 중복 방지
  - 덜 익은 토마토 개수(`cnt`)를 카운팅하여 최종 검증
- **구현 포인트**:
  - 4방향 탐색: `[(1, 0), (0, 1), (-1, 0), (0, -1)]`
  - `while q:` - 큐가 빌 때까지 반복 (❌ `while deque:` 아님!)
  - 새로 익은 토마토를 큐에 추가하면서 `cnt` 감소
  - 최종적으로 `cnt == 0`이면 성공, 아니면 -1 출력

### [11723] 집합 ✅
- **알고리즘**: 집합(Set) 자료구조
- **문제점 및 해결**:
  - ❌ **메모리 초과**: 모든 명령어를 리스트에 저장 `cmd_list = [input().split() for _ in range(n)]`
  - ✅ **해결**: 한 줄씩 처리하여 메모리 O(1)로 개선
  - ❌ **시간 초과**: 함수 호출 오버헤드 + `input()` 느림
  - ✅ **해결**: `sys.stdin.readline` 사용 + 함수 제거
- **최적화 포인트**:
  - `import sys; input = sys.stdin.readline` - 입력 속도 2~3배 향상 🚀
  - 함수 호출 제거 - 직접 처리로 약 30% 빠름
  - `s.discard()` 사용 - `remove()`와 달리 없어도 에러 안남
  - `set(range(1, 21))` - 반복문보다 빠른 all 구현
  - `match-case` → `if-elif` - 더 빠르고 호환성 좋음

## 배운 점

1. **BFS 패턴**
   - 최단 거리/최소 시간 문제는 BFS가 정답
   - 초기 상태를 모두 큐에 넣고 시작하는 패턴
   - 레벨별로 탐색하여 자동으로 최소값 보장

2. **백준 최적화 필수 테크닉** ⭐
   - `sys.stdin.readline` 사용 - 입력 많으면 **필수!**
   - 리스트에 모든 입력 저장하지 말고 **한 줄씩 처리**
   - 불필요한 함수 호출 제거 - 직접 처리가 더 빠름
   - `s.discard()` > `s.remove()` - 안전하고 빠름
   - `set(range())` > `for` 반복문 - 집합 생성 최적화

3. **Python 문법**
   - `is` vs `==`: `is`는 객체 동일성, `==`는 값 비교
   - `while q:` - 큐가 비었는지 체크 (빈 컬렉션은 False)
   - `while deque:` - 클래스 자체는 항상 True (버그!)
   - `match-case` - Python 3.10+만 지원 (백준에서는 if-elif 추천)

4. **디버깅 팁**
   - PowerShell에서 숫자 입력 시 오류는 명령어로 인식되기 때문
   - Python 실행: `python 파일.py` 또는 `python 파일.py < input.txt`
   - Python 인터프리터 빠져나오기: `exit()` 또는 `Ctrl+Z`
