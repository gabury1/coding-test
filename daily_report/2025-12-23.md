# 2025-12-23 코딩테스트 문제 풀이

## 해결한 문제

### [2667] 단지번호붙이기
- **알고리즘**: DFS (깊이 우선 탐색)
- **핵심 개념**:
  - 방향 벡터를 사용한 4방향 탐색
  - 재귀를 통한 연결된 영역 탐색
  - 방문 처리로 중복 탐색 방지
- **개선 사항**:
  - 함수명: `search` → `dfs` (명확성 향상)
  - 4방향 탐색을 반복문으로 처리하여 코드 중복 제거
  - 변수명 개선: `result` → `complexes`, `r` → `count`

### [7576] 토마토
- **알고리즘**: BFS (너비 우선 탐색)
- **문제점 및 해결**:
  - ❌ **시간초과 원인**: DFS + 매 루프마다 전체 그리드 탐색
  - ✅ **해결**: BFS로 변경하여 각 칸을 한 번씩만 방문
- **핵심 개념**:
  - BFS는 최단 거리/최소 시간 문제에 적합
  - `collections.deque` 사용으로 O(1) 시간에 push/pop
  - 큐에 (y, x, day)를 저장하여 날짜 추적
  - 방문 처리로 중복 방지
- **시간복잡도**:
  - 기존: O((N×M)² × 날짜)
  - 개선: O(N×M)

## 배운 점

1. **DFS vs BFS 선택 기준**
   - DFS: 연결된 영역 찾기, 경로 탐색
   - BFS: 최단 거리, 최소 시간 문제

2. **효율성**
   - 매번 전체 탐색은 시간초과 위험
   - 방문 체크로 중복 처리 방지 필수

3. **코드 스타일**
   - 방향 벡터 사용으로 코드 간결화
   - 명확한 변수명으로 가독성 향상
